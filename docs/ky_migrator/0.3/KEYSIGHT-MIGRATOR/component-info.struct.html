COMPONENT-INFO [Public Structure]

COMPONENT-INFO  

Package: keysight-migrator (Nick name(s): :kym)

Common data for model component records.

⚠️ This structure and its derived structures has two states:

  • In-session state where all sysids are handles of type [entity]
  • Persisted (resolved) state where all sysids are the sysid strings retrieved from the [entity] handles that were previously there. Use the component-info-persisted-p predicate to determine if the component-info object has been resolved.

Slots

sysid {[entity] | string}
Initially an entity handle. After conflict resolution the (unique) component sysid.
typename {string}
The user friendly name of the component type.
filename {pathname}
File name of the component in the staging area.
child-sysids {string list}
The sysids of the structure children (filing contents clusters).
tags {keyword list}

Additional classification keywords. Currently in use are:

  • :inseparable-master
  • :inseparable-child

Related functions

Name Summary
component-info-persisted-p Predicate to determine if a component-info object has been persisted and contains actual sysids rather than [entity] handles.
